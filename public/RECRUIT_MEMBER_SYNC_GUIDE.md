# Recruit ↔ Member 양방향 동기화 가이드

## 📋 개요
Potential 및 Serious 등급의 멤버는 자동으로 Recruit 리스트와 동기화됩니다.

## 🔄 자동 동기화 기능

### 1. **멤버 → Recruit 자동 생성**
- 멤버의 상태를 `Potential` 또는 `Serious`로 변경하면 자동으로 Recruit 리스트에 추가됩니다
- 동기화 항목: 이름, 전공, 직업, 직장, 관계, 만난 시기, 생년월일, 나이, 성별, 적합도, 기본성향 평가

### 2. **양방향 실시간 동기화**
- Recruit에서 정보 수정 → 연결된 멤버 정보도 자동 업데이트
- 멤버에서 정보 수정 → 연결된 Recruit 정보도 자동 업데이트
- 상담 히스토리도 양방향 동기화

### 3. **삭제 동기화**
- Recruit 삭제 → 연결된 Potential/Serious 멤버도 함께 삭제
- Potential/Serious 멤버 삭제 → 연결된 Recruit도 함께 삭제
- 일반 멤버로 변경(New(Code-in) 등) → Recruit 리스트에서 자동 제거

## 📊 적합도 자동 계산 시스템

### 기본성향 평가 항목
1. **관계 점수** (1~10점)
2. **Market** 
   - L (Large): 10점
   - M (Medium): 8점
   - S (Small): 6점
3. **체크리스트** (각 10점)
   - ✓ 결혼
   - ✓ 자녀
   - ✓ 주택
   - ✓ 인컴
   - ✓ 야심
   - ✓ 불만족
   - ✓ PMA
   - ✓ 기업가형

### 자동 계산
- 위 항목들의 합계가 자동으로 **적합도 점수**로 계산됩니다 (최대 100점)
- 평가 항목 변경 시 실시간으로 재계산

## 🎯 사용 방법

### Recruit에서 멤버로 승급
1. Recruit 탭에서 대상자 선택
2. **[멤버]** 버튼 클릭
3. 자동으로 트리에 `New(Code-in)` 상태로 추가
4. 모든 히스토리와 정보가 함께 이동

### 멤버를 Potential/Serious로 관리
1. 멤버 정보 탭에서 상태를 `Potential` 또는 `Serious`로 변경
2. 자동으로 Recruit 리스트에 표시
3. 양쪽에서 동시 관리 가능

### 적합도 평가
1. **Recruit 탭** 또는 **멤버 정보 탭**에서 **[📊 평가]** 버튼 클릭
2. 평가 항목 입력
3. 적합도 자동 계산
4. 양쪽 탭에서 동기화된 점수 확인

## ⚠️ 주의사항
- Potential/Serious 멤버를 삭제하면 Recruit에서도 함께 삭제됩니다
- Recruit를 삭제하면 연결된 멤버도 함께 삭제됩니다
- 멤버 상태를 다른 등급으로 변경하면 Recruit에서 자동 제거됩니다

## 📈 등급별 기본 점수
- **Serious**: 75점 (초기값)
- **Potential**: 60점 (초기값)
- 기본성향 평가를 통해 정확한 점수로 업데이트 가능
